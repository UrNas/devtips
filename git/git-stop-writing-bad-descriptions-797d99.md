# Git: Stop Writing Bad Descriptions

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, commit-messages, descriptive-commits, clear-writing, code-documentation, git-best-practices, maintainable-history

## Steps

### 1. These commit messages say nothing

```bash
$ git log --oneline -4
abc123d fix
def456e update
ghi789f changes
jkl012m stuff
```

### 2. Ask: what problem does this solve?

```bash
# Bad: vague action
fix bug

# Good: specific problem
Fix username validation error
```

### 3. Be specific about the change

```bash
# Bad: generic
Update styles

# Good: targeted
Increase button padding for mobile
```

### 4. Include the why when unclear

```bash
$ git commit -m "Remove unused imports" -m "Reduces bundle size by 15KB"
[main x9y8z7w] Remove unused imports
```

### 5. Future you will understand instantly

```bash
$ git log --oneline -3
x9y8z7w Remove unused imports
a1b2c3d Fix username validation error
d4e5f6g Increase button padding for mobile
```
