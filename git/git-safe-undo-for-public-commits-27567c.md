# Git: Safe Undo for Public Commits

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, revert, public, commits, safe, undo, history, push

## Steps

### 1. Already pushed? Use revert, not reset! ⚠️

```bash
$ git log --oneline -3
a1b2c3d Bad feature that breaks prod
d4e5f6g Add user authentication
g7h8i9j Fix navbar styling
```

### 2. Revert creates opposite commit

```bash
$ git revert a1b2c3d
[main b7c8d9e] Revert 'Bad feature'
 3 files changed, 15 deletions(-)
```

### 3. History preserved, changes undone

```bash
$ git log --oneline -3
b7c8d9e Revert 'Bad feature that breaks prod'
a1b2c3d Bad feature that breaks prod
d4e5f6g Add user authentication
```

### 4. Safe to push - no history rewrite

```bash
$ git push origin main
To origin
 * [new commit]  b7c8d9e..HEAD
```
