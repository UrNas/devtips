# Git: Write Commits Like Commands

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, commit-messages, imperative-mood, writing-style, best-practices, git-conventions, professional-commits

## Steps

### 1. This verb tense makes commits perfect

```bash
$ git commit -m "Fix login validation bug"
[main a1b2c3d] Fix login validation bug
```

### 2. Wrong: describes what you did

```bash
$ git log --oneline -3
def456e Fixed the login bug
ghi789f Added new feature
jkl012m Updated documentation
```

### 3. Right: use imperative mood

```bash
# Think: "If applied, this commit will..."

Fix login validation bug
Add user search feature
Update API documentation
```

### 4. Git itself uses imperative mood

```bash
$ git merge feature-branch
Merge branch 'feature-branch'
$ git revert HEAD
Revert "Add broken feature"
```

### 5. Your commits match Git's style

```bash
$ git log --oneline -4
a1b2c3d Fix login validation bug
e4f5g6h Add user search feature
i7j8k9l Update API documentation
m0n1o2p Merge branch 'feature-auth'
```
