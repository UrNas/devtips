# Git: Keep History Linear & Clean

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, rebase, linear-history, pull-rebase, force-with-lease, clean-history

## Steps

### 1. Rebase before pushing keeps it clean

```bash
$ git pull --rebase origin main
First, rewinding head...
Applying: your feature commit
Successfully rebased and updated refs
```

### 2. Configure rebase as default pull

```bash
$ git config pull.rebase true
```

### 3. Compare messy vs clean history

```bash
# Messy merge history
*   Merge branch 'feature'
|\  
| * Feature commit
* | Another merge

# Clean rebase history
* Feature commit
* Previous commit
* Base commit
```

### 4. Force-with-lease protects shared work

```bash
$ git push --force-with-lease
Enumerating objects: 5, done.
To origin/feature-branch
   a1b2c3d..e4f5g6h  HEAD -> main
```
