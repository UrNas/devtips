# Git: Team Rebase Guidelines

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, team-workflow, rebase-rules, collaboration, merge-strategy, shared-branches

## Steps

### 1. Never rebase shared/public branches

```bash
# DON'T rebase these:
git rebase main  # ❌ main is shared
git rebase develop  # ❌ team branch

# DO rebase these:
git rebase main feature-x  # ✅ private
```

### 2. Rebase feature branches before merge

```bash
$ git checkout feature-auth
$ git rebase main
Successfully rebased and updated refs
```

### 3. Use merge for team integration

```bash
$ git checkout main
$ git merge --no-ff feature-auth
Merge made by the 'ort' strategy.
```

### 4. Result: clean feature, preserved context

```bash
$ git log --oneline --graph -4
*   a1b2c3d Merge branch 'feature-auth'
|\  
| * d4e5f6g Add authentication
| * g7h8i9j Add login form
*   j1k2l3m Previous merge
```
