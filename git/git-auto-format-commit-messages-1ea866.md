# Git: Auto-Format Commit Messages

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, commit-templates, automation, git-config, workflow-optimization, consistent-commits, productivity

## Steps

### 1. Never write commit templates manually

```bash
$ git config --global commit.template ~/.gitmessage
```

### 2. Create your commit message template

_~/.gitmessage_

```bash
# ~/.gitmessage
type(scope): short description

# Why this change?

# What does it fix/add?

# Closes #issue
```

### 3. Now git commit opens your template

```bash
$ git commit
# Opens editor with template pre-filled
```

### 4. Add helpful reminders in comments

_~/.gitmessage_

```bash
# Types: feat, fix, docs, style, refactor, test
# Scope: api, ui, auth, db, config
# Keep first line under 50 characters
# Use imperative mood: "Fix" not "Fixed"
```

### 5. Perfect commits every single time

```bash
$ git log --oneline -3
a1b2c3d feat(auth): add password reset
d4e5f6g fix(ui): center modal on mobile
g7h8i9j docs: update deployment guide
```
