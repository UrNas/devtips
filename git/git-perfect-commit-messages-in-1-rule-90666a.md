# Git: Perfect Commit Messages in 1 Rule

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, commit-messages, conventional-commits, best-practices, team-workflow, clean-history, git-log

## Steps

### 1. This format makes commits crystal clear

```bash
$ git commit -m "feat: add user search"
[main 7a8b9cd] feat: add user search
 1 file changed, 12 insertions(+)
```

### 2. Bad commit messages confuse everyone

```bash
$ git log --oneline -3
abc123d fixed stuff
def456e updates
ghi789f changes
```

### 3. Use conventional commits format

```bash
type(scope): description

feat: new feature
fix: bug fix
docs: documentation
style: formatting
```

### 4. Examples that tell the full story

```bash
$ git commit -m "fix(auth): handle expired tokens"
[main 1a2b3cd] fix(auth): handle expired tokens
$ git commit -m "docs: update API examples"
[main 4d5e6fg] docs: update API examples
```

### 5. Your team will thank you forever

```bash
$ git log --oneline -3
1a2b3cd fix(auth): handle expired tokens
4d5e6fg docs: update API examples
7h8i9jk feat(search): add fuzzy matching
```
