# Git: When to Merge vs Rebase

**Category:** `git`  ·  **Format:** `shorts`

**Tags:** git, merge, rebase, workflow, branching, version-control, collaboration

## Steps

### 1. Feature complete? Use merge to preserve history

```bash
$ git merge feature/user-auth
Merge made by the 'ort' strategy.
 src/auth.js | 45 +++++++++++++++++++++
 1 file changed, 45 insertions(+)
```

### 2. Work in progress? Rebase to stay current

```bash
$ git rebase main
First, rewinding head to replay your work
Applying: add login validation
Applying: fix authentication bug
```

### 3. Merge creates history nodes

```bash
$ git log --oneline --graph -5
*   a1b2c3d Merge branch 'feature'
|\  
| * d4e5f6g Add new feature
* | g7h8i9j Fix main bug
|/  
```

### 4. Rebase creates linear history

```bash
$ git log --oneline -5
j1k2l3m Add new feature
m4n5o6p Fix main bug
p7q8r9s Previous commit
s1t2u3v Another commit
```
